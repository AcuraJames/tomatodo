# Ручное тестирование v0.1.12

## 1. Просроченные задачи
1. Создай задачу с датой **вчера**
2. Проверь: тег даты должен быть **красный** (`#ef4444`), не зелёный
3. Проверь в списке «Сегодня» — задача должна появиться там
4. Создай задачу с датой **3 дня назад** — тоже красный, тоже в «Сегодня»

## 2. Break не убивается при сворачивании
1. Запусти фокус (любая длительность)
2. Дождись окончания или пропусти до break
3. Во время отдыха **сверни вкладку** на 2-3 секунды
4. Вернись — таймер отдыха должен продолжать тикать, статус — `running`

## 3. Создание задачи из таймера
1. На странице таймера нажми `+` (новая задача)
2. Введи название, проект **не выбирай** (оставь «Входящие»)
3. Сохрани — задача должна появиться в селекте и в списке «Входящие»
4. Проверь: projectId задачи — не `'inbox'`, а пустая строка

## 4. Браузерные уведомления
1. Разреши уведомления в браузере
2. Запусти фокус на 1 минуту
3. По окончании должно появиться уведомление «Фокус завершён»
4. По окончании break — «Перерыв окончен»

## 5. Восстановление таймера
1. Запусти фокус на 25 минут
2. Подожди 10 секунд
3. **Перезагрузи страницу** (F5)
4. Таймер должен показывать ~24 мин 50 сек, статус — `idle`

## 6. 404 маршрут
1. Перейди по несуществующему URL, например `/asdfgh`
2. Должен произойти редирект на `/`

## 7. Юнит-тесты
```bash
npm test
```
Все 52 теста должны быть зелёными.

## 8. Shared-утилиты
Убедись что теги дат и проектов отображаются корректно:
1. В таймере (выбор задачи) — теги проекта и даты внутри селекта
2. В списках задач (inbox/today/week) — теги проекта и даты
3. В задачах проекта — тег даты есть, тега проекта нет
